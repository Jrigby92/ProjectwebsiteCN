2.1.7 Describe which are the appropriate filetypes for websites

There are many file types which are compatible with websites, to define wether or not any are appropriate is wether or not they are compatible with the rest of your content and also how practical they are in terms of size etc. In my website I have found that .html, .css, .mp4, and .jpg files have been perfectly helpful and compatible with my goals. I have found further information online that provides more detail on these and other file types:

.HTML/ .HTM — HyperText Markup Language file HTML is Hypertext Markup Language, the format used for web pages. It is both a language and a file format. HTML files are text-based, and can be viewed in a plain-text editor. When looking at a .html file in a text editor, you will see markup — characters that do not appear when the same document is viewed in web browser, and which provide information about formatting and style. HTML is the language of the web — every website you view has HTML behind the scenes. For this reason (among others), many languages and applications have tools for outputting HTML. [2]

CSS — Cascading StyleSheet CSS files are a tool in the repertoire of webmasters that take care of how their websites look. CSS files can be created or edited in any text-editor, like Notepad. [1]

JPG/ .JPEG — Joint Photographic Experts Group file Another very common image file format, mainly used for photos[1]

.MOV/ .QT — QuickTime MOVie The QuickTime format was designed by Apple and originated on the Mac, but has made the transition to the PC and is hugely popular.[1]

.pdf PDF is the Portable Document Format. This is a format pioneered by Adobe and useful for fix-format document sharing. Unlike .doc and other types of application-specific formats, a .pdf file contains everything needed to render the document — fonts, images, layout, paper size. The PDF format is intended to display exactly the same way on in any environment. PDFs can be generated in a number of ways. Adobe Acrobat is the standard application for editing and authoring PDFs. There are also tools for printing directly to PDF from other programs (treating the PDF-generator as a printer). Some software utilities also output PDF. The standard application for viewing PDFs is the Adobe Acrobat Reader, though alternatives exist — the Preview app on Macs can view PDFs, and Windows 10 includes Reader, a PDF-viewing app. [2]
