I would argue that for a linear project such as this one, with a relatively small time scale and which is being completed by a single person, less time could be put into the organisation and objective tracking of the project.

Thus allowing more time to complete the more vital tasks such as covering the unit objectives and planning and designing the website.

Alternative to my own approach, perhaps it would aid productivity to create a basic webpage html shell and replicate it for each unit point. Then when each point is complete and pasted onto the websites html shell; edits, stylisation and functionality could be added in a global way, whilst adding slight tweaks at the very of the project for each webpage. This approach would have allowed for better time management, and would enable me or a project team leader to know how much time Is left to finalise the project.

Additionally, I believe a proper analysis of all of the project questions at the beginning would have made it more clear from the outset, and easier to write each unit as the tasks progressed. This would have also allowed me to plan the website more efficiently in advance.
