3.4 Describe how you would go
about testing digital solutions.




























For my project I utilised several digital solutions in creating the website, for now I will focus on how developing coding techniques offered me solutions to several problems throughout the project.


I will focus on my coding for this question, because it’s the digital solution which required the most testing.

Being quite new to HTML and CSS, I experimented often with coding techniques and with almost every step and new function / div group etc which I created, I found it was worth testing. Sometimes I discovered changes that I made would have undesirable effects and the testing process helped me learn. I carried out testing by editing code in visual studio editor and then using google chrome to see how those edits affected the final webpage. I also used safari on occasion to double check the website was stable across browsers. In the event that I would find an error, I would go back into studio editor and try using different techniques until I got the code perfect. Sometimes this involved researching new techniques online to find the most appropriate ones, editing the code and testing again. I continued this process throughout the development of the website.

On one occasion I made 10 or 12 changes without testing, before having to undo all of them and restart that webpage to get things working again, this led me to the conclusion that testing is an absolutely vital step that should be continually carried out throughout the build process. Not only to ensure quality control but also to enhance the learning experience of any project.
